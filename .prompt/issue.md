



Triết lý cốt lõi: FE & BE là 2 hệ thống độc lập hoàn toàn

FE hoàn toàn có thể hoạt động không cần BE 
Khi API BE sẵn sàng -> mới thay thế sau


-> Ý tưởng:
+ FE không biết BE tồn tại 
+ FE giao tiếp với Mapper 
+ Mapper nhận thông tin từ api đổ về FE 

+ Mapper bổ sung hook (seeddataa)
 + Trong trường hợp BE chưa sẵn sàng API 
 + Sử dụng hook để thao tác 100% qua Local Storage
